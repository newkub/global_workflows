## Goal

ปรับปรุง user experience, flows, และ user-centered design ให้เป็นไปตามหลักการ UX ที่ดี

## Scope

ปรับปรุง UX สำหรับ web applications และ mobile interfaces ครอบคลุม user research, information architecture, และ UX patterns ทั้งหมด

## Execute

### 1. Research And Discovery

วิจัยและค้นหาข้อมูล user

1. ทำ `user research` ด้วย interviews และ surveys
2. สร้าง `user personas` และ `user journeys`
3. วิเคราะห์ `user flows` ปัจจุบัน
4. ระบุ `pain points` ในการใช้งาน
5. วิเคราะห์ `competitors` และ industry standards

### 2. Information Architecture

ออกแบบโครงสร้างข้อมูล

1. สร้าง `site maps` และ `content hierarchies`
2. ออกแบบ `navigation structures`
3. จัดระเบียบ `content categorization`
4. สร้าง `taxonomy` และ `tagging systems`
5. วางแผน `search architecture`

### 3. Wireframing And Prototyping

สร้าง wireframes และ prototypes

1. สร้าง `low-fidelity wireframes`
2. สร้าง `high-fidelity mockups`
3. สร้าง `interactive prototypes`
4. ทำ `user testing` กับ prototypes
5. ปรับปรุง designs ตาม feedback

### 4. Design UX Flow

ออกแบบ user flow ที่ลื่นไหล

1. สร้าง `user journey mapping` สำหรับทุก scenarios
2. วาง `flow diagrams` สำหรับทุก interactions
3. กำหนด `state transitions` ที่ชัดเจน
4. ลด `cognitive load` ด้วย simplification
5. เพิ่ม `progressive disclosure` สำหรับ complex features

### 5. Improve UX

ปรับปรุง user experience

1. ทำให้ `user flows` ง่ายขึ้น
2. ลดจำนวน `clicks` ที่จำเป็น
3. เพิ่ม `feedback` สำหรับ user actions
4. ปรับปรุง `navigation`
5. เพิ่ม `accessibility` ตาม `WCAG`

### 6. Test UX

ทดสอบ UX

1. ทดสอบ `usability` กับ users
2. ทดสอบ `accessibility` ด้วย tools
3. ทดสอบ `cross-browser compatibility`
4. ทดสอบ `responsive design`
5. รวบรวม `feedback` และปรับปรุง

### 7. Analytics And Iteration

วิเคราะห์และปรับปรุงซ้ำ

1. ติดตั้ง `analytics tracking`
2. วิเคราะห์ `user behavior data`
3. ติดตาม `key metrics` (conversion, retention, engagement)
4. ทำ `A/B testing` สำหรับ improvements
5. ปรับปรุง designs ตาม data-driven insights

## Rules

### 1. User-Centered Design

ออกแบบโดยเน้น user เป็นศูนย์กลาง

- เข้าใจ `user personas` และ `user journeys`
- ออกแบบตาม `user needs` ไม่ใช่ preferences
- ทำให้งานง่ายที่สุดเท่าที่จะทำได้
- ให้ `feedback` ชัดเจนสำหรับทุก actions

### 2. Research And Discovery

วิจัยและค้นหาข้อมูลก่อนออกแบบ

- ทำ `user research` ด้วย interviews และ surveys
- วิเคราะห์ `competitors` และ industry standards
- สร้าง `user personas` ที่มีพื้นฐานจาก data
- ระบุ `pain points` จาก user feedback
- ใช้ `data-driven decisions` ไม่ใช่ assumptions

### 3. Information Architecture

ออกแบบโครงสร้างข้อมูลที่ชัดเจน

- สร้าง `site maps` ที่ logical
- จัดระเบียบ `content hierarchies` ที่เข้าใจง่าย
- ออกแบบ `navigation structures` ที่ intuitive
- สร้าง `taxonomy` และ `tagging systems` ที่ consistent
- วางแผน `search architecture` ที่ efficient

### 4. Wireframing And Prototyping

สร้าง wireframes และ prototypes ก่อน implement

- เริ่มจาก `low-fidelity wireframes` ก่อน
- สร้าง `high-fidelity mockups` หลัง approve wireframes
- สร้าง `interactive prototypes` สำหรับ testing
- ทำ `user testing` กับ prototypes
- ปรับปรุง designs ตาม feedback ก่อน implement

### 5. Accessibility

ให้ทุกคนใช้งานได้

- ตรงตาม `WCAG guidelines`
- ใช้ `semantic HTML`
- ให้ `keyboard navigation` ทำงานได้
- ใช้ `ARIA labels` ที่เหมาะสม
- ตรวจสอบ `color contrast`

### 6. Mobile First

ออกแบบสำหรับ mobile ก่อน

- ออกแบบสำหรับ `small screens` ก่อน
- ทำให้ `touch targets` มีขนาดเพียงพอ
- ปรับปรุง `mobile performance`
- ทดสอบบน `devices` หลากแบบ

### 7. Analytics And Iteration

วิเคราะห์และปรับปรุงซ้ำตาม data

- ติดตั้ง `analytics tracking` สำหรับ key events
- วิเคราะห์ `user behavior data` อย่างสม่ำเสมอ
- ติดตาม `key metrics` (conversion, retention, engagement)
- ทำ `A/B testing` สำหรับ improvements
- ปรับปรุง designs ตาม data-driven insights

## Expected Outcome

- `User experience` ดีขึ้นอย่างชัดเจน
- `Accessibility` ตรงตามมาตรฐาน
- `User satisfaction` เพิ่มขึ้น
- `User flows` ลื่นไหลและ intuitive
