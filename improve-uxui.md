## Goal

ปรับปรุง UX/UI ให้ดีขึ้นตาม design system principles และ best practices

## Scope

ใช้สำหรับปรับปรุง UX/UI ของ web applications, mobile apps, และ desktop applications

## Execute

### 1. Improve Design System

ปรับปรุง design system

1. ทำ `/idea-uxui` เพื่อวิเคราะห์และสร้างไอเดีย UX/UI improvements
2. ทำ `/follow-web-design` สำหรับ design system principles
3. สร้าง design tokens (colors, spacing, typography)
4. สร้าง component library ที่ reusable
5. ใช้ spacing, typography, colors ที่สอดคล้องกัน
6. สร้าง design guidelines ที่ชัดเจน

### 2. Improve Accessibility

ปรับปรุง accessibility

1. ทำ `/improve-accessibility` เพื่อปรับปรุง accessibility ครบวงจร

### 3. Improve UX Writing

ปรับปรุง UX writing

1. เขียนข้อความที่ชัดเจนและเข้าใจง่าย
2. ใช้ภาษาที่เป็นมิตรกับผู้ใช้
3. หลีกเลี่ยงคำศัพท์เทคนิคที่ไม่จำเป็น
4. ใช้ active voice แทน passive voice
5. สร้าง microcopy ที่ช่วยสนับสนุน user journey
6. ทำตาม UX writing best practices

### 4. Improve Performance

ปรับปรุง performance ของ UI

1. ทำ `/improve-web-performance` เพื่อปรับปรุง web performance ครบวงจร

## Rules

### 1. Design System First

ใช้ design system ก่อน

- สร้าง design tokens ก่อน implement
- ใช้ component library ที่ reusable
- ทำตาม design guidelines ที่กำหนด
- รักษาความสม่ำเสมอทั่วทั้ง application

### 2. Accessibility First

ให้ความสำคัญกับ accessibility

- ทำตาม WCAG standards
- ใช้ semantic HTML
- ให้ keyboard navigation ที่ดี
- ให้ screen reader support
- ใช้ colors ที่มี contrast ดี

### 3. Performance Matters

ให้ความสำคัญกับ performance

- Optimize assets ก่อน deploy
- ใช้ lazy loading สำหรับ heavy components
- ตรวจสอบ Core Web Vitals
- ลด unnecessary re-renders
- ใช้ caching อย่างเหมาะสม

## Expected Outcome

- UX/UI ดีขึ้นอย่างชัดเจน
- Design system ที่ scalable และ maintainable
- Accessibility ตรงตามมาตรฐาน
- Performance ดีขึ้น
- User satisfaction สูงขึ้น
- Consistent design ทั่วทั้ง application
