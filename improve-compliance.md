## Goal

ปรับปรุง regulatory compliance และ industry standards ให้ผ่าน

## Scope

ใช้สำหรับปรับปรุย compliance ทั้ง GDPR, SOC2, HIPAA, PCI DSS, CCPA

## Execute

### 1. Check GDPR Compliance

ตรวจสอบและปรับปรุย GDPR compliance

- ตรวจสอบ data processing records
- ตรวจสอบ consent management
- ตรวจสอบ data subject rights
- ตรวจสอบ data breach notification
- แก้ไข gaps ที่พบ

### 2. Check SOC2 Compliance

ตรวจสอบและปรับปรุย SOC2 compliance

- ตรวจสอบ security controls
- ตรวจสอบ availability controls
- ตรวจสอบ processing integrity
- ตรวจสอบ confidentiality controls
- แก้ไข gaps ที่พบ

### 3. Check HIPAA Compliance (ถ้าจำเป็น)

ตรวจสอบและปรับปรุย HIPAA compliance

- ตรวจสอบ PHI protection
- ตรวจสอบ access controls
- ตรวจสอบ audit controls
- ตรวจสอบ transmission security
- แก้ไข gaps ที่พบ

### 4. Check Industry-Specific Regulations

ตรวจสอบและปรับปรุย industry-specific regulations

- ตรวจสอบ PCI DSS (ถ้าจำเป็น)
- ตรวจสอบ CCPA (ถ้าจำเป็น)
- ตรวจสอบ industry-specific standards
- ตรวจสอบ regional regulations
- แก้ไข gaps ที่พบ

### 7. Run Compliance Audit

รัน compliance audit

## Rules

### 1. GDPR Requirements

ทำตาม GDPR requirements

- ให้ consent อย่างชัดเจน
- ให้ data subject rights
- ให้ data breach notification ภายใน 72 ชั่วโมง
- ให้ data processing records

### 2. SOC2 Requirements

ทำตาม SOC2 requirements

- ให้ security controls
- ให้ availability controls
- ให้ processing integrity
- ให้ confidentiality controls

### 3. Data Privacy Principles

ทำตาม data privacy principles

- Data minimization
- Purpose limitation
- Storage limitation
- Accuracy and integrity

### 4. Priority-Based Fixes

แก้ไขตามลำดับความสำคัญ

- Critical: non-compliance รุนแรง
- High: compliance gaps สำคัญ
- Medium: documentation ขาด
- Low: เป็น compliance optimizations

## Expected Outcome

- GDPR compliance ผ่าน
- SOC2 compliance ผ่าน (ถ้าจำเป็น)
- Industry regulations ผ่าน
- Data privacy ถูกคุ้มครอง
- Audit trail ครบถ้วน
- Compliance gaps ถูกระบุและแก้ไข
