## Goal

เทียบเทียบกับ competitors หรือสิ่งที่คล้ายกัน และสร้างไอเดีย features พร้อม continuous numbering

## Scope

สร้างไอเดีย features ทั้ง enhancements (1-40) และ new features (41-60) พร้อม continuous numbering

## Execute

### 1. Planning And Preparation

วางแผนการเปรียบเทียบและสร้างไอเดียก่อนเริ่มทำจริง

1. ทำ `/plan` เพื่อวางแผนการเปรียบเทียบและสร้างไอเดียครบถ้วน
2. รอยืนยันจาก user ก่อนเริ่มทำ

### 2. Compare And Analyze

วิเคราะห์ competitors หรือ products ที่คล้ายกัน

1. วิเคราะห์ competitors หรือ products ที่คล้ายกัน
2. สรุป features ที่เขามีแต่เราไม่มี
3. ระบุ gaps และ differentiation opportunities
4. ศึกษา user feedback จาก reviews/forums

### 3. Generate Enhancement Ideas (1-40)

สร้างไอเดียปรับปรุง features ที่มีอยู่

1. สร้างไอเดียปรับปรุง features ที่มีอยู่ 40 อันดับ
2. Numbering เริ่มที่ 1
3. ระบุ Parent Feature ที่ชัดเจน

### 4. Generate New Feature Ideas (41-60)

สร้างไอเดีย features ใหม่

1. สร้างไอเดีย features ใหม่ 20 อันดับ
2. Numbering ต่อจาก enhancements (41-60)
3. ระบุ Category และ Priority

### 5. Create Unified Report

สร้างตารางรวม (Enhancements 1-40 + New Features 41-60)

| Order | Type | Description | Improve/Problem | Parent/Category | Priority |
|-------|------|-------------|-----------------|---------------|----------|
| 1 | Enhancement | ... | ... | Authentication | 🔴/🟡/🟢 |
| ... | ... | ... | ... | ... | ... |
| 40 | Enhancement | ... | ... | ... | ... |
| 41 | New Feature | ... | ... | ... | 🔴/🟡/🟢 |
| ... | ... | ... | ... | ... | ... |
| 60 | New Feature | ... | ... | ... | 🔴/🟡/🟢 |

- Type: Enhancement หรือ New Feature
- Priority: 🔴 Critical, 🟡 Recommended, 🟢 Optional
- แยกตารางหรือรวมเป็นตารางเดียวก็ได้ ขอให้ numbering continuous

## Rules

### 1. Numbering Continuous

- Enhancements: 1-40
- New Features: 41-60
- ไม่เริ่ม 1 ใหม่สำหรับ New Features

### 2. Enhancements First

- แสดง enhancements (1-40) ก่อน
- ตามด้วย new features (41-60)
- ให้เห็นว่าเป็นการพัฒนาต่อยอดจากของเดิม

### 3. Consistent Formatting

- ใช้ตารางเดียวหรือแยกก็ได้ แต่ numbering ต้องต่อกัน
- ระบุ Type (Enhancement/New Feature) ชัดเจน
- Priority levels ต้องสอดคล้องกัน

## Expected Outcome

- ตาราง 60 อันดับ (1-40 Enhancements, 41-60 New Features)
- Continuous numbering ไม่เริ่มใหม่
- Enhancements อยู่ด้านบน
- New Features อยู่ด้านล่าง
- พร้อม Roadmap และ Priority Justification
