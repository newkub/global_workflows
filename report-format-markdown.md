## Goal

จัดรูปแบบ markdown documents ให้สอดคล้องกับมาตรฐานและอ่านง่าย

## Scope

ใช้สำหรับการจัดรูปแบบ output จาก:
- Documentation
- README files
- Change logs
- Technical reports

## Execute

### 1. Define Document Structure

1. กำหนด hierarchy ของ headings (H1-H6)
2. ใช้ H1 สำหรับ document title
3. ใช้ H2-H3 สำหรับ main sections
4. ใช้ H4-H6 สำหรับ subsections

### 2. Format Content

1. ใช้ bold สำหรับ keywords สำคัญ
2. ใช้ italic สำหรับ emphasis
3. ใช้ code blocks สำหรับ code snippets
4. ใช้ backticks สำหรับ inline code

### 3. Add Lists and Tables

1. ใช้ unordered lists สำหรับ bullet points
2. ใช้ ordered lists สำหรับ numbered steps
3. ใช้ tables สำหรับ tabular data
4. ใช้ task lists สำหรับ checklists

### 4. Add Links and Images

1. ใช้ markdown links สำหรับ URLs
2. ใช้ image syntax สำหรับ images
3. ใช้ reference-style links สำหรับ links ที่ซ้ำกัน
4. ใช้ alt text สำหรับ accessibility

## Rules

### Heading Structure

โครงสร้าง headings ที่ต้องใช้

- ใช้ H1 สำหรับ document title (เพียง 1 ครั้ง)
- ใช้ H2 สำหรับ main sections
- ใช้ H3-H6 สำหรับ subsections
- ไม่ข้าม heading levels

### Content Formatting

การจัดรูปแบบเนื้อหา

- ใช้ bold สำหรับ keywords สำคัญ
- ใช้ italic สำหรับ emphasis
- ใช้ code blocks พร้อม language tags
- ใช้ backticks สำหรับ inline code

### Lists and Tables

การใช้ lists และ tables

- ใช้ unordered lists สำหรับ bullet points
- ใช้ ordered lists สำหรับ numbered steps
- ใช้ tables พร้อม headers ชัดเจน
- ใช้ alignment ที่เหมาะสมกับ data types

### Links and Images

การใช้ links และ images

- ใช้ descriptive link text
- ใช้ alt text สำหรับ images
- ใช้ reference-style links สำหรับ links ที่ซ้ำกัน
- ตรวจสอบว่า links ใช้งานได้

## Expected Outcome

- Markdown document ที่อ่านง่าย
- Heading structure ที่ชัดเจน
- Content formatting ที่ consistent
- Links และ images ที่ถูกต้อง
