## Goal

ดูแลรักษาโปรเจกต์ให้มีความมั่นคงด้วย compliance และ maintenance ที่เป็นระบบ

## Execute

### 1. Compliance

1. รักษา compliance ตามกฎหมาย
2. ปกป้องความเป็นส่วนตัวของข้อมูล
3. รักษา security compliance
4. ปฏิบัติตาม accessibility standards
5. จัดการ licensing อย่างถูกต้อง
6. รักษา audit trails
7. ประเมินความเสี่ยงอย่างสม่ำเสมอ

### 2. Maintenance

1. อัพเดท dependencies อย่างสม่ำเสมอ
2. จัดการ technical debt
3. วางแผน refactoring อย่างเป็นระบบ
4. จัดการ legacy code
5. Deprecate features อย่างถูกต้อง
6. ตรวจสอบสุขภาพระบบอย่างสม่ำเสมอ
7. ตอบสนอง alerts อย่างรวดเร็ว

## Rules

1. ปฏิบัติตาม compliance frameworks
2. บันทึก compliance measures
3. ใช้ automated dependency updates
4. ให้ความสำคัญกับการลด technical debt
5. วางแผน maintenance windows
6. แจ้งให้ stakeholders ทราบ

## Expected Outcome

- Regulatory compliance
- Reduced technical debt
- Well-maintained codebase
- Proactive issue resolution
