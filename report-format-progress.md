## Goal

แสดงความคืบหน้าของงานหรือ process ให้ชัดเจน

## Scope

ใช้สำหรับการแสดง:
- Task progress
- Build status
- Deployment progress
- Batch operations

## Execute

### 1. Track Progress

1. ระบุ total items และ completed items
2. ระบุ current step และ remaining steps
3. ระบุ elapsed time และ estimated time
4. ระบุ blockers และ issues

### 2. Format Progress

1. ใช้ progress bars สำหรับ visual indicators
2. ใช้ percentages สำหรับ quick overview
3. ใช้ step lists สำหรับ detailed progress
4. ใช้ status indicators (✅, ⏳, ❌)

### 3. Add Context

1. เพิ่ม timestamps สำหรับ reference
2. เพิ่ม ETA สำหรับ planning
3. เพิ่ม blockers พร้อม solutions
4. เพิ่ม next steps และ dependencies

### 4. Highlight Status

1. ใช้ colors สำหรับ status (green, yellow, red)
2. ใช้ bold สำหรับ current step
3. ใช้ icons สำหรับ status changes
4. ใช้ summaries สำหรับ quick overview

## Rules

### Visual Indicators

ตัวบ่งชี้แบบ visual

- ใช้ progress bars สำหรับ percentages
- ใช้ consistent colors สำหรับ status
- ใช้ clear labels สำหรับ steps
- ใช้ icons สำหรับ status changes

### Accuracy

ความถูกต้องของข้อมูล

- ใช้ real-time updates
- ใช้ accurate percentages
- ใช้ realistic ETAs
- ใช้ current blockers

### Context

บริบทที่ควรมี

- timestamps สำหรับ reference
- ETAs สำหรับ planning
- blockers พร้อม solutions
- next steps และ dependencies

## Expected Outcome

- Progress ที่ชัดเจนและ accurate
- Visual indicators ที่เป็นประโยชน์
- Context ที่เพียงพอ
- Status ที่ up-to-date
