## Goal

เขียนเนื้อหาให้ชัดเจน ชัดแจ้ง ไม่กำกวม และเข้าใจง่าย

## Scope

ใช้สำหรับเขียน workflows, skills, documentation, และ code comments

## Execute

### 1. Follow Content Quality

ทำ `/follow-content-quality` สำหรับคุณภาพเนื้อหาพื้นฐาน

### 2. Write Explicitly

เขียนให้ชัดเจนและชัดแจ้ง

1. ใช้ภาษาที่ตรงไปตรงมา ไม่ใช้คำที่กำกวม
2. ระบุ subject และ object ชัดเจนในทุกประโยค
3. หลีกเลี่ยง pronouns ที่ไม่ชัดเจน (it, they, this) ถ้าไม่มี context
4. ใช้ active voice แทน passive voice
5. ระบุเงื่อนไขและข้อจำกัดชัดเจน
6. ให้ตัวอย่าง concrete แทน abstract descriptions

### 3. Avoid Ambiguity

หลีกเลี่ยงความกำกวมในการเขียน

1. หลีกเลี่ยงคำที่มีหลายความหมาย (should, could, might)
2. ใช้คำที่เฉพาะเจาะจงแทนคำทั่วไป
3. ระบุ exceptions และ edge cases ชัดเจน
4. แยกประเด็นที่แตกต่างออกจากกัน
5. ใช้ numbered list สำหรับ steps ที่ต้องทำตามลำดับ

### 4. Validate Clarity

ตรวจสอบความชัดเจน

1. อ่านซ้ำและถามตัวเองว่า "มีความหมายอื่นไหม"
2. ตรวจสอบว่า reader จะเข้าใจเหมือนกันหรือไม่
3. ตรวจสอบว่าไม่มี assumptions ที่ไม่ได้ระบุ
4. ตรวจสอบว่ามี context เพียงพอหรือไม่

## Rules

- ใช้ภาษาที่ตรงไปตรงมาเสมอ
- ระบุ subject และ object ชัดเจนในทุกประโยค
- หลีกเลี่ยงคำที่กำกวมและ pronouns ที่ไม่ชัดเจน
- ใช้ active voice แทน passive voice
- ระบุเงื่อนไขและข้อจำกัดชัดเจน
- ให้ตัวอย่าง concrete แทน abstract descriptions
- หลีกเลี่ยงคำที่มีหลายความหมาย
- ใช้คำที่เฉพาะเจาะจงแทนคำทั่วไป
- ระบุ exceptions และ edge cases ชัดเจน
- แยกประเด็นที่แตกต่างออกจากกัน
- ใช้ numbered list สำหรับ steps ที่ต้องทำตามลำดับ
- ตรวจสอบความชัดเจนด้วยการอ่านซ้ำ
- ตรวจสอบว่าไม่มี assumptions ที่ไม่ได้ระบุ

## Expected Outcome

- เนื้อหาที่ชัดเจน ชัดแจ้ง ไม่กำกวม
- Reader เข้าใจเหมือนกัน
- ไม่มี assumptions ที่ไม่ได้ระบุ
- ลดความสับสนและการตีความผิด
