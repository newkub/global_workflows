## Goal

เขียน best practices จาก official documentation และเอกสารที่เป็นทางการ

## Scope

ใช้สำหรับเขียน best practices สำหรับ library, framework, หรือ technology ที่มี official documentation

## Execute

### 1. Research Best Practices

ค้นหาและเรียนรู้ best practices จาก sources ที่เป็นทางการ

1. ทำ `/follow-learn-best-practices` หรือ `/deep-research`
2. อ่าน `official documentation` และสรุป best practices
3. บันทึก `sources` ทั้งหมด

### 2. Write Best Practices

เขียน best practices ตามมาตรฐานที่เรียนรู้มา

1. ระบุไฟล์ที่ต้องการแก้ไขใน `project`
2. แก้ไขไฟล์เดิมให้สอดคล้องกับ best practices ที่เรียนรู้มา
3. ทำตาม `/follow-write-workflows` สำหรับการเขียน

## Rules

### 1. Source Validation

ตรวจสอบความถูกต้องของ sources

- ใช้ official documentation เป็น primary source
- ตรวจสอบว่า sources เป็นปัจจุบัน
- บันทึก URLs และ references ทั้งหมด
- หลีกเลี่ยง sources ที่ไม่เป็นทางการ

### 2. Application Guidelines

ใช้ best practices อย่างถูกต้อง

- ทำ `/follow-learn-best-practices` เป็น primary workflow
- แก้ไขไฟล์เดิม ไม่สร้างไฟล์ใหม่
- เขียนเป็นภาษาไทย
- ทำตาม `/follow-write-workflows` สำหรับการเขียน

### 3. Documentation Standards

บันทึกและ document อย่างเป็นระบบ

- บันทึก sources ทั้งหมด
- เขียน documentation สำหรับ future reference
- อัพเดท references เมื่อมีการเปลี่ยนแปลง
- ใช้ `/update-reference` เมื่อมี file operations

## Expected Outcome

- Best practices ที่ถูกต้องและเป็นปัจจุบัน
- ไฟล์ที่แก้ไขแล้วสอดคล้องกับ best practices
- Documentation ที่บันทึกไว้สำหรับ future reference
- Sources ที่ถูกต้องและเป็นทางการ
