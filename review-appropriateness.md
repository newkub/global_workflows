## Goal

ตรวจสอบให้หน่อยว่าส่วนนี้เหมาะสมไหม เหมาะสมพอดี ไม่น้อยหรือมากเกินไป

## Execute

### 1. Analyze Appropriateness

1. ตรวจสอบความเหมาะสมของส่วนที่ระบุ
2. เปรียบเทียบกับ best practices และ patterns ที่ใช้
3. ประเมินว่า scope เหมาะสมหรือไม่
4. ตรวจสอบว่าไม่ซับซ้อนเกินไปหรือไม่

### 2. Validate Balance

1. ตรวจสอบว่าไม่น้อยเกินไป (under-engineered)
2. ตรวจสอบว่าไม่มากเกินไป (over-engineered)
3. ตรวจสอบว่าเหมาะสมพอดีกับ use case
4. ประเมิน trade-offs ที่เกิดขึ้น

## Rules

### 1. Analysis

ตรวจสอบความเหมาะสมอย่างเคร่งครัด

- เปรียบเทียบกับ patterns และ best practices
- พิจารณา context และ use case
- ประเมิน impact ที่เกิดขึ้น

### 2. Balance

ตรวจสอบความสมดุลของการออกแบบ

- หลีกเลี่ยง under-engineering
- หลีกเลี่ยง over-engineering
- หาจุดสมดุลที่เหมาะสม

### 3. Feedback

ให้ feedback ที่ชัดเจนและ action-oriented

- ระบุปัญหาที่พบ
- ให้คำแนะนำที่เฉพาะเจาะจง
- ระบุ priority สำหรับการแก้ไข

## Expected Outcome

- ส่วนนี้เหมาะสมพอดี
- ไม่น้อยหรือมากเกินไป
