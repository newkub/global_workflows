## Goal

ปรับปรุง SEO ให้ website มี ranking ที่ดีขึ้นใน search engines

## Execute

### 1. Analyze SEO

วิเคราะห์ SEO ปัจจุบัน

1. รัน SEO audit tools ด้วย `/unlighthouse`
2. ตรวจสอบ current ranking
3. วิเคราะห์ competitors
4. ระบุ `keywords` ที่สำคัญ
5. ตรวจสอบ `technical SEO` issues

### 2. Optimize Content

ปรับปรุง content สำหรับ SEO

1. เพิ่ม `keywords` ที่เหมาะสม
2. ปรับปรุง `meta tags` (`title`, `description`)
3. ใช้ `heading structure` ที่ถูกต้อง
4. เพิ่ม `internal links`
5. ปรับปรุง `content quality`

### 3. Optimize Technical SEO

ปรับปรุง technical SEO

1. ปรับปรุง `site speed`
2. ใช้ `sitemap.xml`
3. ใช้ `robots.txt` ที่เหมาะสม
4. ใช้ `schema markup`
5. แก้ไข `broken links`

### 4. Optimize UX Signals

ปรับปรุง signals ที่ search engines ใช้

1. ปรับปรุง `mobile experience`
2. ลด `bounce rate`
3. เพิ่ม `time on site`
4. ปรับปรุง `CTR`
5. เพิ่ม `user engagement`

### 5. Build Authority

สร้าง authority สำหรับ website

1. สร้าง `backlinks` ที่มีคุณภาพ
2. เพิ่ม `social signals`
3. สร้าง `content` ที่มีคุณค่า
4. เข้าร่วมใน `communities`
5. ทำ `guest posting`

### 6. Monitor SEO

ตรวจสอบและวัดผล SEO

1. ติดตาม `ranking` อย่างต่อเนื่อง
2. ตรวจสอบ `traffic` และ `conversions`
3. วิเคราะห์ `user behavior`
4. ตรวจสอบ competitors
5. ปรับปรุงตามข้อมูล

## Rules

### Keyword Strategy

ใช้ keywords อย่างมีประสิทธิภาพ

- วิจัย `keywords` ที่มี volume สูงและ competition ต่ำ
- ใช้ `long-tail keywords`
- กระจาย `keywords` อย่างเหมาะสม
- หลีกเลี่ยง `keyword stuffing`
- ใช้ `keywords` ใน natural context

### Technical SEO

ปรับปรุง technical aspects

- ตรวจสอบ `page speed`
- ใช้ `HTTPS`
- ใช้ `mobile-friendly design`
- ใช้ `clean URL structure`
- แก้ไข `duplicate content`

### Content Quality

สร้าง content ที่มีคุณภาพ

- เขียน `content` ที่มีประโยชน์ต่อ users
- ใช้ `multimedia` อย่างเหมาะสม
- อัปเดต `content` อย่างสม่ำเสมอ
- ใช้ `internal linking`
- ปรับปรุง `readability`

### User Experience

SEO ต้องเน้น UX

- ลด `bounce rate`
- เพิ่ม `time on page`
- ปรับปรุง `navigation`
- ทำให้ site เร็วขึ้น
- ทำให้ `mobile-friendly`

### Analytics

ใช้ data เพื่อตัดสินใจ

- ติดตาม `key metrics`
- วิเคราะห์ `user behavior`
- ทดลอง `A/B testing`
- ปรับปรุงตามข้อมูล
- รายงานผลอย่างสม่ำเสมอ

## Expected Outcome

- `Ranking` ดีขึ้นใน search engines
- `Organic traffic` เพิ่มขึ้น
- `Technical SEO` issues แก้ไขแล้ว
- `Content` มีคุณภาพและ relevant
- `User experience` ดีขึ้น
- `SEO metrics` ดีขึ้นอย่างต่อเนื่อง
- Commit เสร็จสมบูรณ์ตาม `conventional commits`
