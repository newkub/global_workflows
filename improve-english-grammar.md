## Goal

ตรวจสอบและปรับปรุง English grammar ใน code comments, documentation, และ messages ให้ถูกต้องและเป็นมืออาชีพ

## Scope

ใช้สำหรับตรวจสอบ English grammar ใน code comments, documentation files, README, และ user-facing messages

## Execute

### 1. Identify Text To Check

ระบุ text ที่ต้องตรวจสอบ grammar

1. ค้นหา code comments ที่เป็นภาษาอังกฤษ
2. ค้นหา documentation files (README, docs/)
3. ค้นหา user-facing messages (error messages, UI text)
4. ค้นหา commit messages ที่เป็นภาษาอังกฤษ

### 2. Check Grammar Errors

ตรวจสอบ grammar errors อย่างเป็นระบบ

1. ตรวจสอบ subject-verb agreement
2. ตรวจสอบ tense consistency
3. ตรวจสอบ article usage (a, an, the)
4. ตรวจสอบ preposition usage
5. ตรวจสอบ sentence structure

### 3. Improve Clarity

ปรับปรุงความชัดเจนของข้อความ

1. ทำให้ประโยคกระชับและเข้าใจง่าย
2. ใช้ active voice แทน passive voice
3. หลีกเลี่ยง complex sentence structures
4. ใช้ vocabulary ที่เหมาะสมกับ context
5. ตรวจสอบ consistency ของ terminology

### 4. Verify Changes

ตรวจสอบผลลัพธ์หลังปรับปรุง

1. ตรวจสอบว่า meaning ไม่เปลี่ยนแปลง
2. ตรวจสอบว่า grammar ถูกต้อง
3. ตรวจสอบว่า consistency ถูกรักษา
4. รัน linting เพื่อตรวจสอบ code ยังทำงานได้

## Rules

### 1. Grammar Fundamentals

ตรวจสอบ grammar fundamentals อย่างเป็นระบบ

- Subject-verb agreement ต้องถูกต้องเสมอ
- Tense consistency ในแต่ละ paragraph
- Article usage (a, an, the) ตาม rules
- Preposition usage ตาม standard English
- Pluralization ตาม rules ของ nouns

### 2. Clarity And Conciseness

ปรับปรุงความชัดเจนและความกระชับ

- ใช้ active voice แทน passive voice เมื่อเป็นไปได้
- ทำให้ประโยคสั้นและเข้าใจง่าย
- หลีกเลี่ยง jargon ที่ไม่จำเป็น
- ใช้ simple words แทน complex words
- หลีกเลี่ยง redundancy และ wordiness

### 3. Context-Specific Guidelines

ทำตาม guidelines เฉพาะของแต่ละ context

- Code comments: ใช้ present tense สำหรับ descriptions
- Documentation: ใช้ clear headings และ structure
- Error messages: ใช้ concise และ actionable language
- UI text: ใช้ consistent terminology
- Commit messages: ใช้ imperative mood

### 4. Consistency Standards

รักษา consistency ทั่วทั้ง codebase

- ใช้ terminology ที่สม่ำเสมอ
- ใช้ capitalization rules ที่สม่ำเสมอ
- ใช้ punctuation style ที่สม่ำเสมอ
- ใช้ spelling variants ที่สม่ำเสมอ (US vs UK)
- ตรวจสอบ glossary หากมี

## Expected Outcome

- English grammar ถูกต้องและเป็นมืออาชีพ
- Text ชัดเจนและเข้าใจง่าย
- Consistency ทั่วทั้ง codebase
- Code comments และ documentation มีคุณภาพสูง

## Common Mistakes (optional)

ข้อผิดพลาดที่พบบ่อย:

- Subject-verb agreement errors
- Tense inconsistency
- Incorrect article usage
- Passive voice overuse
- Complex sentence structures
- Inconsistent terminology
- Spelling variants mixing (US/UK)
