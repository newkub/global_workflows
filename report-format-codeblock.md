## Goal

จัดรูปแบบ code blocks ให้สวยงามและอ่านง่ายด้วย syntax highlighting

## Scope

ใช้สำหรับการจัดรูปแบบ:
- Code snippets
- API documentation
- Configuration examples
- Technical tutorials

## Execute

### 1. Select Language

1. ระบุ programming language สำหรับ code block
2. ใช้ language tags ที่ถูกต้อง (typescript, python, rust, etc.)
3. ตรวจสอบว่า language รองรับ syntax highlighting
4. ใช้ plain text ถ้าไม่มี language ที่เหมาะสม

### 2. Format Code

1. ใช้ proper indentation (2 หรือ 4 spaces)
2. จัด line breaks ให้อ่านง่าย
3. ใช้ consistent code style
4. เพิ่ม comments สำหรับ code ที่ซับซ้อน

### 3. Add Metadata

1. เพิ่ม filename หรือ path ด้านบน code block
2. เพิ่ม line numbers ถ้าจำเป็น
3. เพิ่ม copy button สำหรับ convenience
4. เพิ่ม language indicator

### 4. Highlight Key Parts

1. ใช้ bold หรือ emphasis สำหรับ keywords สำคัญ
2. ใช้ comments สำหรับ explanations
3. ใช้ separators สำหรับแยก sections
4. ใช้ inline annotations ถ้าจำเป็น

## Rules

### Language Selection

การเลือก language ที่เหมาะสม

- ใช้ language tags ที่ถูกต้องและ standard
- ใช้ generic tags สำหรับ languages ที่ไม่รู้จัก
- ตรวจสอบว่า language รองรับ syntax highlighting
- ใช้ plain text ถ้าไม่มี language ที่เหมาะสม

### Code Formatting

การจัดรูปแบบ code

- ใช้ consistent indentation ทั้ง code block
- จัด line breaks ให้อ่านง่าย
- ใช้ consistent code style
- เพิ่ม comments สำหรับ code ที่ซับซ้อน

### Metadata

ข้อมูลเพิ่มเติมที่ควรมี

- filename หรือ path ด้านบน code block
- line numbers สำหรับ debugging
- language indicator ชัดเจน
- copy button สำหรับ convenience

## Expected Outcome

- Code blocks ที่มี syntax highlighting ถูกต้อง
- Code ที่อ่านง่ายและ consistent
- Metadata ที่ครบถ้วน
- Key parts ที่ถูก highlight
