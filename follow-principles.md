## Goal

เขียนเนื้อหาเป็นหลักการทั่วไปตาม principles ที่กำหนด ไม่เจาะจง

## Scope

ใช้สำหรับเขียนทุกอย่าง - code, documentation, workflows, skills, config files

## Execute

### 1. Identify Principles

ระบุ principles ที่เกี่ยวข้อง

1. อ่าน principles จาก `principles/` folder หรือ skill documentation
2. ระบุ principles ที่เกี่ยวข้องกับ task ปัจจุบัน
3. เรียงลำดับ principles ตามความสำคัญ

### 2. Apply Principles

นำ principles มาใช้กับงาน

1. เขียนเป็นหลักการ how-to ทั่วไป
2. ไม่เจาะจง tools หรือ frameworks เฉพาะ
3. ใช้ placeholders หรือ generic terms
4. อ้างอิง examples ที่เข้าใจง่าย

### 3. Validate Principles

ตรวจสอบว่า principles ถูกนำไปใช้

1. ตรวจสอบว่าเนื้อหาไม่เจาะจง
2. ตรวจสอบว่า principles ถูกปฏิบัติตาม
3. ทำ `/improve-content` เพื่อปรับปรุงคุณภาพ content

## Rules

### 1. Generic Over Specific

เขียนเป็นหลักการทั่วไป ไม่เจาะจง

- ใช้ generic terms แทน specific tools
- ใช้ placeholders สำหรับ values
- อธิบาย concepts ไม่ใช่ implementations
- หลีกเลี่ยง hard-coded values

### 2. Principle-Based Writing

เขียนตาม principles ที่กำหนด

- อ่าน principles ก่อนเขียน
- นำ principles มาใช้ทุกขั้นตอน
- ตรวจสอบว่า principles ถูกปฏิบัติตาม
- อ้างอิง principles ที่ใช้

### 3. How-To Format

เขียนเป็นหลักการ how-to

- อธิบายวิธีการทำงาน
- ไม่เขียนเป็น manual step-by-step
- ใช้ bullet points สำหรับรายการ
- มี examples ที่เข้าใจง่าย

### 4. Consistency

รักษาความสม่ำเสมอ

- ใช้คำศัพท์สม่ำเสมอ
- ใช้ structure สม่ำเสมอ
- ใช้ format สม่ำเสมอ
- ตรวจสอบความสอดคล้อง

### 5. Clarity

เขียนให้ชัดเจน

- ใช้ภาษาที่เข้าใจง่าย
- หลีกเลี่ยงคำที่กำกวม
- มี examples ที่ชัดเจน
- อธิบาย concepts อย่างถูกต้อง

## Expected Outcome

- เนื้อหาเป็นหลักการทั่วไป
- Principles ถูกนำไปใช้
- ไม่เจาะจง tools หรือ frameworks
- เข้าใจง่ายและใช้งานได้จริง
- ผ่านการตรวจสอบ /improve

## Common Mistakes (optional)

ข้อผิดพลาดที่พบบ่อย

- เขียนเจาะจง tools หรือ frameworks
- ไม่อ่าน principles ก่อนเขียน
- เขียนเป็น manual step-by-step
- ใช้ hard-coded values
- ไม่มี examples ที่ชัดเจน
