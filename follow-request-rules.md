## Goal

ขอ rules ที่อธิบายวิธีการและแนวทางในการเขียนผลลัพธ์ที่ต้องการ

## Execute

### 1. Analyze Desired Output

1. วิเคราะห์ผลลัพธ์ที่ต้องการ
2. ระบุลักษณะเฉพาะของผลลัพธ์
3. หาโครงสร้างและรูปแบบที่ต้องการ
4. ระบุ constraints และ requirements

### 2. Request Rules

1. ขอ rules ที่อธิบายวิธีการเขียน
2. ขอแนวทางและ best practices
3. ขอขั้นตอนและเงื่อนไข
4. ขอตัวอย่างและกรณีศึกษา

### 3. Validate Rules

1. ตรวจสอบว่า rules สอดคล้องกับผลลัพธ์ที่ต้องการ
2. ทดสอบ rules กับกรณีตัวอย่าง
3. ตรวจสอบว่า rules ครอบคลุมทุกกรณี
4. ยืนยันว่า rules นำไปใช้ได้จริง

### 4. Apply Rules

1. ประยุกต์ใช้ rules กับงานจริง
2. ตรวจสอบผลลัพธ์ที่ได้
3. ปรับปรุง rules ถ้าจำเป็น
4. บันทึก rules ที่ใช้ได้ผล

## Rules

1. Rules ต้องอธิบายวิธีการอย่างชัดเจน
2. ต้องมีตัวอย่างประกอบ
3. ต้องครอบคลุมทุกกรณีที่เป็นไปได้
4. ต้องสอดคล้องกับผลลัพธ์ที่ต้องการ
5. ห้ามขอ rules ที่ซับซ้อนเกินความจำเป็น

## Expected Outcome

- Rules ที่อธิบายวิธีการอย่างชัดเจน
- แนวทางและ best practices
- ตัวอย่างและกรณีศึกษา
- Rules ที่นำไปใช้ได้จริง
