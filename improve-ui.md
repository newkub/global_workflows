## Goal

ปรับปรุง user interface, design system, visual hierarchy, motion และ animation ให้เป็นไปตามหลักการ UI ที่ดี

## Scope

ปรับปรุง UI สำหรับ web applications และ mobile interfaces ครอบคลุม design systems, component libraries, และ visual patterns ทั้งหมด

## Execute

### 1. Design System Setup

ตั้งค่า design system

1. สร้าง `design tokens` (colors, typography, spacing)
2. สร้าง `component library`
3. กำหนด `design principles` และ guidelines
4. สร้าง `documentation` สำหรับ design system
5. ตั้งค่า `version control` สำหรับ designs

### 2. Improve UI

ปรับปรุง user interface

1. ใช้ `design system` ที่สอดคล้องกัน
2. ปรับปรุง `visual hierarchy`
3. ใช้ `spacing` และ `typography` ที่เหมาะสม
4. เพิ่ม `micro-interactions`
5. ปรับปรุง `color contrast`

### 3. Design Motion And Animation

ออกแบบ motion และ animation ที่เป็นธรรมชาติ

1. ใช้ `animation principles` (easing, timing, choreography)
2. ออกแบบ `transitions` ที่ smooth และ meaningful
3. เพิ่ม `loading states` ด้วย animations
4. ใช้ `micro-animations` สำหรับ feedback
5. ปรับปรุง `performance` ของ animations

### 4. Optimize Performance

ปรับปรุง performance ของ UI

1. ลด `initial load time`
2. เพิ่ม `lazy loading` สำหรับ images
3. ใช้ `CSS animations` แทน JavaScript
4. ปรับปรุง `rendering performance`
5. เพิ่ม `loading states`

### 5. Test UI

ทดสอบ UI

1. ทดสอบ `cross-browser compatibility`
2. ทดสอบ `responsive design`
3. ทดสอบ `accessibility` ด้วย tools
4. รวบรวม `feedback` และปรับปรุง

## Rules

### 1. Design System

ใช้ design system ที่ consistent และ scalable

- สร้าง `design tokens` (colors, typography, spacing)
- สร้าง `component library` ที่ reusable
- กำหนด `design principles` และ guidelines
- สร้าง `documentation` สำหรับ design system
- ตั้งค่า `version control` สำหรับ designs

### 2. Consistency

รักษาความสม่ำเสมอใน design

- ใช้ `design system` เดียวกัน
- ใช้ `spacing`, `typography`, `colors` ที่สอดคล้องกัน
- ใช้ `patterns` ที่คุ้นคนคุ้นล้ำ
- รักษา `visual hierarchy` ที่ชัดเจน

### 3. Performance

UI ต้องเร็วและ responsive

- ตั้งเป้าหมาย `load time` ที่รับได้
- ใช้ `lazy loading` สำหรับ assets
- ปรับปรุง `rendering performance`
- เพิ่ม `loading states` และ `skeletons`

### 4. Motion And Animation

ใช้ motion และ animation อย่างเหมาะสม

- ใช้ `animation principles` (easing, timing, choreography)
- ออกแบบ `transitions` ที่ smooth และ meaningful
- ใช้ `micro-animations` สำหรับ feedback
- ปรับปรุง `performance` ของ animations
- หลีกเลี่ยง animations ที่สร้าง distraction

### 5. Accessibility

ให้ทุกคนใช้งานได้

- ตรงตาม `WCAG guidelines`
- ใช้ `semantic HTML`
- ให้ `keyboard navigation` ทำงานได้
- ใช้ `ARIA labels` ที่เหมาะสม
- ตรวจสอบ `color contrast`

## Expected Outcome

- `UI` มีความสม่ำเสมอและ professional
- `Design system` ที่ scalable และ maintainable
- `Performance` ดีขึ้น
- `Accessibility` ตรงตามมาตรฐาน
