## Goal

เรียนรู้ best practices จาก official website และเอกสารที่เป็นทางการ เพื่อให้ได้ความรู้ที่ถูกต้องและเป็นปัจจุบัน และนำไปปรับใช้กับไฟล์ที่แนบให้

## Execute

### 1. Identify Best Practices Source

1. ระบุ library, framework หรือ tool ที่ต้องการเรียนรู้ best practices
2. หา official website และ documentation
3. ตรวจสอบว่ามี DeepWiki หรือ Context7 สำหรับ source นี้หรือไม่

### 2. Use Learn From Web

1. ใช้ `/learn-from-web` สำหรับ libraries และ frameworks
2. ใช้ DeepWiki สำหรับ GitHub repositories
3. ใช้ Context7 สำหรับ library documentation
4. ใช้ Web Search เป็น fallback
5. เข้าถึง Official Documentation เสมอ

### 3. Use Deep Research

1. ใช้ `/deep-research` เมื่อต้องการข้อมูลลึกจาก multiple sources
2. ค้นหาจาก Google, NPM, GitHub
3. ใช้ DeepWiki สำหรับ GitHub repositories
4. ใช้ Context7 สำหรับ library documentation
5. Synthesize findings จากทุก sources

### 4. Apply Best Practices to Attached Files

**ถ้า learn แล้วให้แก้ไขไฟล์ที่แนบให้ถาม best practices**

1. อ่านไฟล์ที่แนบให้ทั้งหมด
2. วิเคราะห์ว่าไฟล์เหล่านี้ต้องการปรับปรุงอะไรตาม best practices
3. แก้ไขไฟล์ตาม best practices ที่เรียนรู้มา
4. เขียน best practices ที่สำคัญลงในไฟล์ documentation
5. ระบุ patterns และ conventions ที่แนะนำ
6. เขียน code examples ที่ดี
7. เขียน configuration examples ที่เหมาะสม
8. เขียน common pitfalls และวิธีหลีกเลี่ยง
9. สร้าง summary สำหรับแต่ละ best practice

### 5. Validate And Apply

1. ทดลองใช้งานตาม best practices
2. สร้างโปรเจกต์ตัวอย่างเพื่อทดสอบ
3. เปรียบเทียบกับ alternatives
4. นำ best practices ไปใช้ในโปรเจกต์จริง
5. บันทึก lessons learned

## Rules

- ใช้ `/learn-from-web` เป็น primary workflow สำหรับ learning
- ใช้ `/deep-research` เมื่อต้องการข้อมูลลึกจาก multiple sources
- ลำดับความสำคัญ: Official Docs → DeepWiki → Context7 → Web Search
- ตรวจสอบว่าข้อมูลเป็นปัจจุบัน
- ทดสอบความเข้าใจด้วยโปรเจกต์ตัวอย่าง
- เปรียบเทียบ best practices จากหลายแหล่ง
- บันทึก sources ทั้งหมด
- **ถ้า learn แล้วให้แก้ไขไฟล์ที่แนบให้ถาม best practices**

## Expected Outcome

- Best practices ที่ถูกต้องและเป็นปัจจุบัน
- Code examples และ configurations ที่ดี
- ความเข้าใจที่ถูกต้องเกี่ยวกับ library/framework
- การนำความรู้ไปใช้งานจริง
- Documentation ที่บันทึกไว้สำหรับ future reference
- ไฟล์ที่แนบให้ถูกแก้ไขตาม best practices
