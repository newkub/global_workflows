## Goal

สร้างหรือจัดระเบียน skills ในโครงสร้างที่ถูกต้องตามมาตรฐาน write-skills

## Execute

### 1. Prepare

1. อ่าน write-skills SKILL.md
2. ตรวจสอบประเภท skill ที่เหมาะสม
3. ศึกษาโครงสร้างไดเรกทอรีที่กำหนด

### 2. Research

1. อ่าน skills/write-skills/skills-types/README.md
2. เลือกประเภท skill ที่เหมาะสมกับงาน
3. ดูตัวอย่างโครงสร้างของแต่ละประเภท

### 3. Analyze

1. ระบุว่า skill ที่จะสร้างเป็นประเภทใด
2. กำหนดไฟล์ที่จำเป็นต้องมี
3. วางแผนเนื้อหาของแต่ละไฟล์

### 4. Plan

1. สร้างโครงสร้างไดเรกทอรีตามประเภท
2. กำหนดลำดับการเขียนไฟล์
3. เตรียม verification checklist

### 5. Execute

1. สร้างไดเรกทอรีตามที่กำหนด
2. เขียน SKILL.md หลัก
3. สร้างไฟล์ย่อยตามประเภท
4. เขียนเอกสารตามโครงสร้างที่กำหนด
5. ใช้ active voice
6. ใส่ตัวอย่างที่ชัดเจน

### 6. Verify

1. ตรวจสอบว่าโครงสร้างไฟล์ถูกต้องตามประเภท
2. ตรวจสอบว่าทุกไฟล์ที่จำเป็นมีอยู่
3. ยืนยันว่าผ่าน write-rules ทั้งหมด
4. อ่านทวนเอกสารทั้งหมด
5. ตรวจสอบความชัดเจนของภาษา
6. ยืนยันว่ามีตัวอย่างที่เพียงพอ

### 7. Finalize

1. สรุปสิ่งที่สร้างหรือแก้ไข
2. ยืนยัน skill พร้อมใช้งาน
3. รัน `/update-reference` สำหรับไฟล์ที่เพิ่ม

## Rules

### 1. Structure

- ใช้โครงสร้างตามประเภท skill ที่เลือก
- สร้างไฟล์ที่จำเป็นทั้งหมดตามประเภท
- จัดเรียงไฟล์ตามลำดับที่เหมาะสม

### 2. Content

- เขียนเอกสารตามโครงสร้างที่กำหนด
- ใช้ active voice
- ใส่ตัวอย่างที่ชัดเจน
- เขียนกระชับ ตรงประเด็น

### 3. Verification

- ตรวจสอบว่าโครงสร้างไฟล์ถูกต้องตามประเภท
- ตรวจสอบว่าทุกไฟล์ที่จำเป็นมีอยู่
- ยืนยันว่าผ่าน write-rules ทั้งหมด

### 4. Connected Workflows

- ใช้ /write-workflows สำหรับการเขียน workflow
- ใช้ /write-skills สำหรับการสร้าง skills

## Expected Outcome

- Skill ที่สร้างมีโครงสร้างถูกต้องตามประเภทที่เลือก
- เอกสารครอบคลุมทุกด้านที่จำเป็น
- ผ่าน verification checklist ทั้งหมด
- พร้อมใช้งานจริง
