## Goal

Review documentation layer ทั้งหมมเพื่อระบุและแก้ไข issues ตาม priority

## Scope

ครอบคลุม documentation completeness, accuracy, code comments, API docs

## Execute

### 1. Analyze Documentation

1. ตรวจสอบ README files
2. Review API documentation
3. Check code comments
4. Validate guides and tutorials

### 2. Review Completeness

1. Check documentation coverage
2. Review getting started guides
3. Validate API docs completeness
4. Check examples coverage

### 3. Review Accuracy

1. Validate documentation accuracy
2. Check code examples
3. Review outdated information
4. Validate consistency

### 4. Review Quality

1. Check documentation clarity
2. Review documentation structure
3. Validate formatting
4. Check language and grammar

### 5. Prioritize And Report

1. Group issues ตาม impact
2. ใช้ `/report` เพื่อสรุปผล
3. รอ user confirmation

### 6. Execute Fixes

1. Add missing documentation
2. Fix inaccurate information
3. Improve documentation quality
4. Update outdated docs

## Rules

### 1. Documentation Coverage

ตรวจสอบ documentation coverage:

- README ครบถ้วน
- API docs ครบถ้วน
- Code comments ดี
- Guides ครบถ้วน
- Examples มี

### 2. Documentation Accuracy

ตรวจสอบ documentation accuracy:

- Documentation accurate
- Code examples work
- ไม่มี outdated info
- Consistent with code
- Up to date

### 3. Documentation Quality

ตรวจสอบ documentation quality:

- Documentation clear
- Structure good
- Formatting consistent
- Language good
- Easy to understand

### 4. Code Comments

ตรวจสอบ code comments:

- Comments meaningful
- ไม่ comment obvious
- Comments up to date
- ไม่ comment-out code
- Document complex logic

### 5. Documentation Maintenance

ตรวจสอบ documentation maintenance:

- Documentation updated
- Review process มี
- Documentation versioned
- ไม่ stale
- Consistent with releases

## Expected Outcome

- Documentation ครบถ้วน
- Documentation accurate
- Documentation quality ดี
- Code comments ดี
- Documentation up to date

## Common Mistakes

ข้อผิดพลาดที่พบบ่อย:

- Documentation ไม่ครบถ้วน
- Outdated information
- Poor quality
- ไม่มี examples
- Comments ไม่ meaningful

## Anti-Patterns

รูปแบบที่ไม่ควรทำ:

- ❌ Documentation ไม่ครบถ้วน
- ❌ Outdated information
- ❌ Poor quality
- ❌ ไม่มี examples
- ❌ Comments ไม่ meaningful
