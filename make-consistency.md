## Goal

ทำให้สม่ำเสมอกันด้วยหลักการทั่วไปที่ใช้ได้กับทุกอย่าง

## Scope

ใช้สำหรับทำให้สม่ำเสมอในทุกด้าน (code, documentation, workflows, skills, configuration, file structure, naming, patterns, processes, data, UI, UX)

## Execute

### 1. Analyze Current State

วิเคราะห์สถานะปัจจุบันเพื่อหา inconsistency

1. ตรวจสอบ naming, style, structure ทั่วทั้งระบบ
2. ตรวจสอบ duplicate content ทุกประเภท
3. วิเคราะห์ patterns ที่ไม่สม่ำเสมอ
4. ตรวจสอบ configuration ทั้งหมด

### 2. Establish Standards

ตั้งมาตรฐานสำหรับความสม่ำเสมอ

1. ตั้งมาตรฐาน naming conventions ทั่วทั้งระบบ
2. ตั้งมาตรฐาน structure สำหรับทุกอย่าง
3. ตั้งมาตรฐาน patterns ทั่วไป
4. ตั้งมาตรฐาน format ทั่วไป

### 3. Apply Consistency

นำมาตรฐานไปใช้จริง

1. ปรับปรุง naming ทั่วทั้งระบบ
2. ปรับปรุง structure ทั่วทั้งระบบ
3. ปรับปรุง patterns ให้สม่ำเสมอ
4. ปรับปรุง format ให้สม่ำเสมอ
5. ปรับปรุง configuration ให้สม่ำเสมอ

### 4. Verify Consistency

ตรวจสอบว่าสม่ำเสมอแล้ว

1. ตรวจสอบ naming, style, structure อีกครั้ง
2. ตรวจสอบว่าไม่มี duplicate content
3. ตรวจสอบว่า patterns สม่ำเสมอ
4. ตรวจสอบว่าทุกอย่างสม่ำเสมอ

## Rules

### 1. No Redundancy

หลีกเลี่ยงความซ้ำซ้อนในทุกด้าน

- ลบ duplicate content ทุกประเภท
- ใช้ `single source of truth`
- รวมรายละเอียดที่เหมือนกันไว้ที่เดียว
- หลีกเลี่ยงการ copy-paste โดยไม่จำเป็น

### 2. Alignment

ทำให้สอดคล้องกัน ไม่ขัดแย้งกัน

- ใช้ conventions เดียวกันทั่วระบบ
- ไม่มี conflicts ระหว่าง patterns
- สอดคล้องกับ architecture และ design system
- ใช้ terminology ที่สม่ำเสมอ

### 3. Readability

ทำให้อ่านง่าย เข้าใจง่าย

- ใช้ naming ที่ชัดเจนและสื่อความหมาย
- ใช้ structure ที่เป็นระเบียบ
- ใช้ documentation ที่เข้าใจง่าย
- ใช้ format ที่สม่ำเสมอ

### 4. Sorting And Grouping

จัดเรียงและจัดกลุ่มให้ถูกต้อง

- จัดลำดับตามความสำคัญ
- จัดกลุ่มตาม domain หรือ category
- ใช้ hierarchy ที่ชัดเจน
- ใช้ ordering ที่ logical

## Expected Outcome

- ระบบสม่ำเสมอทั้ง naming, style, structure
- ไม่มี content ที่ซ้ำซ้อน
- อ่านง่าย เข้าใจง่าย
- Sorting และ grouping ถูกต้อง
- Patterns และ conventions สม่ำเสมอ
