## Prompt

ใช้ workflow นี้เมื่อต้องการตรวจสอบและประเมินคุณภาพโค้ดด้วยตนเอง

## Execute

1. Syntax and Structure Verification

- ตรวจสอบ syntax errors
- ตรวจสอบ code structure
- ยืนยัน formatting ถูกต้อง
- รัน linters

2. Semantic and Alignment Validation

- ตรวจสอบ logic correctness
- ยืนยันว่า code ตรงกับ requirements
- ตรวจสอบ best practices
- ประเมิน code quality

3. Self Review

- อ่าน code ตัวเอง
- หา potential issues
- ตรวจสอบ edge cases
- Verify test coverage

4. Document Findings

- สรุป issues ที่พบ
- สร้าง TODO items
- Plan improvements
- Track progress

## Rules

1. Honest Assessment

- ไม่ปกปิด issues
- Be critical
- หา areas ที่ปรับปรุงได้
- ยอมรับ mistakes

2. Thoroughness

- ตรวจสอบทุกส่วน
- ไม่รีบ
- Use checklists
- Take notes

3. Continuous Improvement

- เรียนรู้จาก findings
- Apply lessons
- Update practices
- Share insights

## Expected Outcome

- Code quality ถูกประเมิน
- Issues ถูกระบุ
- Improvement plan สร้างแล้ว
- Code quality ดีขึ้น
