## Goal

ตรวจสอบ SEO เพื่อให้ website มี ranking ที่ดีใน search engines

## Execute

### 1. Check Meta Tags

1. ตรวจสอบ title tags ทุกหน้า
2. ตรวจสอบ meta descriptions ทุกหน้า
3. ตรวจสอบ meta keywords (ถ้าจำเป็น)
4. ตรวจสอบ canonical URLs

### 2. Check Structure

1. ตรวจสอบ heading hierarchy (H1-H6)
2. ตรวจสอบ semantic HTML
3. ตรวจสอบ URL structure
4. ตรวจสอบ internal linking

### 3. Check Sitemap

1. ตรวจสอบ sitemap.xml มีอยู่
2. ตรวจสอบ sitemap ถูกต้อง
3. ตรวจสอบ sitemap ถูก submit ไปยัง search engines
4. ตรวจสอบ robots.txt

### 4. Check Structured Data

1. ตรวจสอบ schema.org markup
2. ตรวจสอบ JSON-LD validity
3. ตรวจสอบ rich snippets
4. ใช้ Google Structured Data Testing Tool

### 5. Check Content

1. ตรวจสอบ content quality
2. ตรวจสอบ keyword usage
3. ตรวจสอบ duplicate content
4. ตรวจสอบ content length

### 6. Check Technical SEO

1. ตรวจสอบ page speed
2. ตรวจสอบ mobile-friendliness
3. ตรวจสอบ HTTPS
4. ตรวจสอบ 404 errors

### 7. Run SEO Audit

1. รัน SEO audit tools (Screaming Frog, Ahrefs, SEMrush)
2. ตรวจสอบ SEO score
3. ตรวจสอบ issues ที่พบ
4. แก้ไข issues ที่พบ

## Rules

### 1. Meta Tag Guidelines

- Title: 50-60 characters
- Description: 150-160 characters
- Keywords: relevant และไม่ spam
- Canonical: ชี้ไปที่ page หลัก

### 2. Structure Guidelines

- H1: 1 ตัวต่อหน้า
- Headings: hierarchy ชัดเจน
- URLs: readable และ descriptive
- Internal links: relevant และ natural

### 3. Priority Levels

- Critical: ทำให้ search engines ไม่สามารถ index ได้
- High: ทำให้ ranking แย่มาก
- Medium: ทำให้ ranking แย่เล็กน้อย
- Low: เป็น best practices

## Expected Outcome

- Meta tags ครบถ้วนและถูกต้อง
- Structure ถูกต้องและ semantic
- Sitemap และ robots.txt ถูกต้อง
- Structured data ถูกต้อง
- Content quality สูง
- Technical SEO ผ่าน
