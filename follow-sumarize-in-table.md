## Goal

สรุปเนื้อหา workflow หรือไฟล์ในรูปแบบตารางที่อ่านง่ายและเข้าใจได้ทันที

## Execute

### 1. Read Target File

1. อ่านไฟล์หรือ workflow ที่ต้องการสรุป
2. ระบุ sections หลักทั้งหมด (Goal, Execute, Rules, Expected Outcome)
3. ระบุ sub-sections และ items ทั้งหมด
4. บันทึก code blocks และ configuration examples

### 2. Extract Key Information

1. ดึงหัวข้อและคำอธิบายจาก Goal
2. ดึง steps และ sub-steps จาก Execute
3. ดึง rules ทั้งหมด
4. ดึก expected outcomes ทั้งหมด
5. จัดกลุ่มข้อมูลตาม category

### 3. Create Table Structure

1. สร้าง table สำหรับ Execute steps (Step | รายละเอียด)
2. สร้าง table สำหรับ Expected Outcome (Outcome | รายละเอียด)
3. สร้าง table สำหรับ Rules (Rule | รายละเอียด)
4. ใช้ numbered columns สำหรับลำดับที่ชัดเจน
5. ใช้ grouping สำหรับ related items

### 4. Format Content

1. ใช้ bullet points ภายใน table cells
2. ใช้ code blocks สำหรับ commands และ paths
3. ใช้ backticks สำหรับ keywords
4. จัดเรียงลำดับตามความสำคัญ
5. ย่อรายละเอียดที่ยาวให้กระชับ

### 5. Validate Table

1. ตรวจสอบว่าข้อมูลครบถ้วน
2. ตรวจสอบว่า table อ่านง่าย
3. ตรวจสอบว่าไม่มีข้อมูลซ้ำซ้อน
4. ตรวจสอบว่าลำดับถูกต้อง
5. ตรวจสอบว่า formatting สอดคล้องกัน

## Rules

- ใช้ table format สำหรับการสรุป
- ใช้ numbered columns สำหรับลำดับที่ชัดเจน
- ใช้ bullet points ภายใน table cells
- ใช้ backticks สำหรับ keywords และ commands
- จัดกลุ่มข้อมูลตาม category
- ย่อรายละเอียดที่ยาวให้กระชับ
- ตรวจสอบว่าข้อมูลครบถ้วน
- ใช้ภาษาไทยในรายละเอียด

## Expected Outcome

- เนื้อหาสรุปในรูปแบบตาราง
- ข้อมูลครบถ้วนและอ่านง่าย
- Table structure ชัดเจน
- Formatting สอดคล้องกัน
- ลำดับข้อมูลถูกต้อง
