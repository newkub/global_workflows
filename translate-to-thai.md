## Goal

แปลเนื้อหาจากภาษาอื่นเป็นภาษาไทยให้ถูกต้องและเป็นธรรมชาติ

## Execute

### 1. Analyze Source Content

1. อ่านและวิเคราะห์เนื้อหาต้นทาง
2. ระบุประเภทเนื้อหา (technical, documentation, conversation)
3. ตรวจสอบ context และ tone ที่เหมาะสม
4. ระบุคำศัพท์เทคนิคที่ควรรักษาไว้เป็นภาษาอังกฤษ

### 2. Translate Content

1. แปลเป็นภาษาไทยโดยรักษาความหมายเดิม
2. ใช้ภาษาไทยที่ถูกต้องและเป็นธรรมชาติ
3. รักษา technical terms ที่ควรใช้ภาษาอังกฤษ
4. ปรับ tone ให้เหมาะสมกับประเภทเนื้อหา

### 3. Review And Refine

1. ตรวจสอบความถูกต้องของการแปล
2. ปรับปรุงความลื่นไหลและความเป็นธรรมชาติ
3. ตรวจสอบความสอดคล้องกับ context
4. ตรวจสอบการสะกดคำและไวยากรณ์

## Rules

### 1. Translation Accuracy

- รักษาความหมายเดิมของเนื้อหา
- หลีกเลี่ยงการแปลตรงตัวที่ไม่เป็นธรรมชาติ
- ใช้คำที่เหมาะสมกับ context
- รักษาความละเอียดของข้อมูล

### 2. Technical Terminology

- ไม่แปลคำเทคนิคที่ควรใช้ภาษาอังกฤษ
- ใช้คำศัพท์เทคนิคที่ถูกต้องใน domain นั้นๆ
- รักษาความสอดคล้องกับ industry standards
- อธิบาย technical terms ในภาษาไทยถ้าจำเป็น

### 3. Tone And Style

- ปรับ tone ให้เหมาะสมกับประเภทเนื้อหา
- ใช้ professional tone สำหรับ documentation
- ใช้ conversational tone สำหรับการสนทนา
- รักษา consistency ของ style

### 4. Code And Commands

- ไม่แปล code, commands, หรือ configuration
- รักษา formatting ของ code blocks
- รักษา comments ใน code เป็นภาษาเดิม
- อธิบาย code ในภาษาไทยถ้าจำเป็น

## Expected Outcome

- เนื้อหาภาษาไทยที่ถูกต้องและเป็นธรรมชาติ
- Technical terminology ที่เหมาะสม
- Tone และ style ที่เหมาะสม
- Code และ commands ที่ไม่ถูกแปล
